僕が朝起きる前と起きてからに、Claude Code の自律ループが何をしているか。Prev / Next で 1 日ぶんを追えます。

朝ブリーフから microCMS 下書きまでの 6 ステップ

  1. 01 / 06 · 07:00 launchd が起動
    Mac の launchd（OS 標準のジョブスケジューラ）に仕込んだ com.tsutsumi.daily-brief.plist が発火し、ヘッドレス Claude Code に 「朝のブリーフを実行」 というプロンプトを投入します。僕はまだ寝ています。
  2. 02 / 06 · 並列起動 (calendar + news)
    親エージェント ai-secretary が calendar-watcher（Google Calendar から今日の予定を取得）と news-curator（登録済み RSS を巡回）を 並列 に起動。出力は daily/2026-04-21.md に append されます。
  3. 03 / 06 · ネタ提案 (blog-seed)
    news-curator の出力を受けて blog-seed が起動し、今日書けそうな記事案を 3 本 生成します。各案にはタイトル、想定ペルソナ、リード文が 1 行ずつ。これも同じ daily ノートに追記されて、通知で先頭 3 行がプッシュされます。
  4. 04 / 06 · 著者が選ぶ
    通知を見た僕が ai-secretary に返信する形で 「2 番、A と F で」 のように ネタ番号とペルソナを指定。ここだけが人間の判断で、あとは再び自走します。ペルソナ未指定のまま執筆には入らないのが掟。
  5. 05 / 06 · 執筆 → レビューのループ (最大 3 回)
    blog-writer が drafts/YYYY-MM-DD-slug.md に本文を生成 → blog-reviewer が静的チェッカ + LLM レビューで判定。要修正 なら指摘を添えて writer に差し戻し。3 回回しても残った指摘は僕の判断待ちキューに入ります。
  6. 06 / 06 · 承認 → microCMS に下書き投稿
    「公開 OK」を受けた僕が microCMS に投稿して と言った瞬間、publish-draft.mjs が下書き API に POST。公開ボタンは押さず 下書き状態 で止めます。最終チェックと公開は管理画面で手動。ここが A モードの生命線。
